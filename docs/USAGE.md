# USAGE — リカーショップゴワ 業務管理プラットフォーム ご利用ガイド

> 本ガイドは **株式会社リカーショップゴワ様** スタッフ向けの操作マニュアルです。
> 後和専務、管理部・事務員、営業ドライバー、経理・横断管理者の皆様にお読みいただけます。
>
> 不明点がありましたら、デジライズ茶圓までお気軽にご連絡ください。

---

## はじめに

### 本プラットフォームでできること

- LINE公式アカウントへのお問い合わせを **自動でチケット化**
- お問い合わせの状況（誰が対応中か・期限は・進捗は）を **一目で把握**
- 担当の引き継ぎ漏れを **「次の担当者へのメモ」で防止**
- 経営層・管理部・現場ドライバーがそれぞれ **必要な情報だけ**確認

### システムへのアクセス

| 環境 | URL |
|---|---|
| 本番 | https://gowa-line-mgr.vercel.app/ （予定） |
| デモ・商談用 | https://gowa-line-demo.vercel.app/ |

PC・スマホどちらからでもご利用いただけます。

# 第2章: 後和専務向け

## 2-1. ダッシュボードの見方

ログイン直後の **ホーム画面** に主要KPIが表示されます。

> [スクショ: ダッシュボード全景]

### 表示される指標

| ブロック | 内容 |
|---|---|
| 本日の未対応 | 30分以上未対応のチケット数 |
| 引き継ぎ待ち | 「次の担当者へ」状態のチケット |
| 紐付け待ち | LINE userId が顧客に紐付いていない件数 |
| 完了率（直近7日） | answered + closed_won / 全件 |
| カテゴリ別件数 | 在庫確認・配達変更・クレーム・他 |
| 担当者別ロード | 各人の対応中件数 |

### グラフの読み方

- **青線**: 受信件数（顧客から）
- **緑線**: 完了件数
- **赤線**: 失注件数

クリックすると該当チケット一覧に飛びます。

## 2-2. カンバンビューでの優先度判断

サイドメニュー **「カンバン」** をクリック。

> [スクショ: カンバンビュー]

### 列の意味

```
[新規] → [対応中] → [社内確認] → [回答待ち] → [完了]
```

各カードには以下が表示されます:

- 顧客名（または LINE displayName）
- カテゴリバッジ（注文 / 配達 / クレーム など）
- 担当者アバター
- 最終更新時刻
- 期限（あれば赤字でカウントダウン）

### 優先度判断のコツ

| 状態 | 判断 |
|---|---|
| 30分以上動いていないカードが新規列にある | 担当者にチャット |
| 「社内確認」が3日以上滞留 | 後和専務が直接アサインを変更 |
| クレームカテゴリが赤バッジ | 即座にエスカレ |

## 2-3. 担当者へのアサイン・差し戻し

カードをクリックすると **チケット詳細**が開きます。

> [スクショ: チケット詳細・担当者ドロップダウン展開]

### アサイン手順

1. 詳細画面の **「担当」** ドロップダウンを開く
2. 候補から担当者を選択
3. **「次の担当者へのメモ」** に引き継ぎ事項を入力（必須・500字以内）
4. **「保存」**

> **Note**: 引き継ぎメモを書かないと保存できない仕様です（属人化リスク防止）。

### 差し戻し（再アサイン）

「担当を解除」ボタンで未アサイン状態に戻せます。Google Chat に通知が飛びます。

## 2-4. 経営層エスカレーション

以下の条件で **自動エスカレ**が発動します:

| 条件 | エスカレ先 |
|---|---|
| 新規受信から30分未対応 | 当日担当 → マネージャー |
| 「回答待ち」24時間放置 | マネージャー → 後和専務 |
| クレームカテゴリ即発生 | 担当 → マネージャー & 後和専務 |

エスカレされると、後和専務のダッシュボード上部に **赤い帯**で表示されます。

> [スクショ: エスカレ通知バナー]

手動エスカレも可能: チケット詳細の **「経営層へエスカレ」** ボタン。

## 2-5. マッピング管理

サイドメニュー **「マッピング」** をクリック。

> [スクショ: マッピング管理画面]

LINEで初めてメッセージを送ってきた人は **未紐付け状態** で並びます。

| カラム | 内容 |
|---|---|
| LINE displayName | 顧客が任意設定する表示名 |
| 初回受信 | 最初にメッセージが来た日時 |
| 件数 | 関連チケット数 |
| 直近メッセージ | 最新本文の冒頭60字 |
| AIサジェスト候補 | 上位3件まで顧客マスタから推測 |
| アクション | 紐付け / 候補から選択 / 手動検索 / スキップ |

### ワンクリック紐付け

AI が「この人は **居酒屋ABC** ですね」と提示してくれます。
合っていれば **「この候補で紐付け」** をクリックするだけで完了。

> [スクショ: AI候補表示・紐付けボタン]

> **Note**: 紐付けると、過去のチケットも自動で顧客名が反映されます。

## 2-6. 設定変更（管理者のみ）

サイドメニュー **「設定」** をクリック（kowa / manager / finance ロールのみ表示）。

### 変更可能項目

| 項目 | 内容 |
|---|---|
| エスカレしきい値 | 30分→60分など |
| 通知先Google Chatスペース | 緊急通知の送信先 |
| 営業時間 | 自動応答テンプレの動作時間 |
| 失注理由マスタ | タグの追加・並び替え |
| ロール別権限 | スタッフが見られる範囲 |

> **Warning**: 設定変更は全社員に影響します。慎重に。

# 第3章: 管理部・事務員向け

## 3-1. 一次対応のフロー

LINEからの問い合わせは **自動でチケット化** されます。事務員様は以下のフローで対応します:

```
1. 通知 / カンバンで新着を確認
2. 内容確認
3. 自分で対応 or 適切な担当者へ転送
4. 「次の担当者へのメモ」を記入
5. ステータス更新
```

> [スクショ: カンバン新規列のカード一覧]

## 3-2. LINE/メールからの問合せ受信

### LINEから来た場合

- 自動でチケット化されます
- 顧客が紐付いていれば **顧客名表示**
- 未紐付けなら **「未紐付け」バッジ**が付きます

> [スクショ: 未紐付けバッジ表示]

### メールから来た場合（Phase 1.5以降）

- メールアドレスから自動で顧客マッチング
- 不明な場合は事務員様が手動で顧客選択

## 3-3. チケットへのステータス更新

### 8段階ステータスの使い分け

| ステータス | いつ使う |
|---|---|
| 新規（open） | 受信直後・誰も対応していない |
| 対応中（triaging） | 自分が見始めた |
| 社内確認（internal_check） | 倉庫に在庫確認、配送に空き確認など |
| 仕入確認（supplier_quote） | メーカー・問屋に問い合わせ中 |
| 回答待ち（awaiting_reply） | 顧客への返信を待っている |
| 回答済（answered） | 一次完了 |
| 受注成立（closed_won） | 注文が確定した |
| 失注（closed_lost） | 結局買ってもらえなかった |

> [スクショ: ステータスドロップダウン]

### ステータス更新手順

1. チケット詳細を開く
2. **「ステータス」** をクリックして変更
3. 必要に応じて **コメント**を残す
4. **保存**

## 3-4. 「次の担当者へのメモ」の書き方

担当者を変更するときは **必ず**メモを書きます。

> [スクショ: 担当者変更モーダル・メモ入力欄]

### 良いメモの例

```
顧客から「黒霧島1ケース、明日10時納品希望」と連絡。
在庫は確認済み（倉庫に5ケースあり）。
ドライバーへ配送指示をお願いします。
```

### 悪いメモの例

```
お願いします。
```

### コツ

- **誰が・何を・いつまでに** を明記
- 既に確認済みの事項を共有
- 次の担当が「すぐ動ける」状態にする

## 3-5. 「聞き直しサジェスト」の活用

AI が顧客への返信内容を提案してくれます。

> [スクショ: 聞き直しサジェスト表示]

### 発動シーン

| 顧客メッセージ | サジェスト |
|---|---|
| 「ワインの値段教えて」 | 「数量と必要日を教えてください」 |
| 「すぐ欲しい」 | 「お急ぎですか？」 |
| 「在庫ある？」 | 「ご予算感を教えていただけますか？」 |

### 使い方

1. チケット詳細の **「聞き直しサジェスト」** カードを確認
2. **「LINE返信下書きへコピー」** をクリック
3. LINE管理画面に貼り付けて送信（または編集）

## 3-6. 失注理由の記録

注文に至らなかった場合、失注理由をタグで記録します。

> [スクショ: 失注理由選択モーダル]

| タグ | 意味 |
|---|---|
| 在庫切れ | 商品が無かった |
| タイミング | 納期・スケジュールが合わなかった |
| 価格 | 価格で折り合わなかった |
| 他社購入 | 他から買われた |
| その他 | 自由記述 |

AIが候補を提示するので、合っていれば1クリックで確定。

> **Note**: 失注理由は**経営層が月次で分析**します。正確な記録にご協力をお願いします。

# よくある質問（FAQ）

### Q1. パスワードを忘れた

A. Google ログインを使っているため、パスワードはありません。Google アカウントの復旧をお試しください。

### Q2. ログインできない（許可ドメイン外）

A. メールアドレスが `@gowa58.co.jp` でない可能性があります。後和専務まで申請してください。

### Q3. 通知が来ない

A. ブラウザ通知を許可してください。Chrome の場合: 設定 > プライバシーとセキュリティ > サイトの設定 > 通知 > 本サイトを許可。

### Q4. チケットの担当を変更できない

A. 権限がない可能性があります。後和専務 / マネージャー / 事務員のみ変更可能です。ドライバー様は依頼してください。

### Q5. 「未紐付け」とは何ですか

A. LINE 公式アカウントは、顧客の本名や顧客コードを取得できない仕様です。初めて連絡してきた人は **未紐付け状態**で表示され、スタッフが手動で「この人は◯◯さん」と紐付ける必要があります。

### Q6. AIが提示する候補が違うとき

A. **「手動検索」**から顧客名で検索して選択してください。間違った候補で紐付けても、後から **「紐付け解除」**できます。

### Q7. 過去のLINE履歴は見られますか

A. 本プラットフォーム稼働前の履歴は、LINE公式管理画面のCSVエクスポート → 取込でしか見られません（LINE仕様の制約）。Phase 1.5で取込機能を追加予定です。

### Q8. PUSH配信（こちらから一斉送信）はいつから？

A. Phase 1.5（2026年7月中旬予定）から開始します。それまでは LINE 公式管理画面で一斉送信してください。

### Q9. 操作ミスでデータが消えた

A. 変更履歴は全件記録されています。後和専務またはデジライズまでご連絡ください。1営業日以内に復旧可能です。

### Q10. スマホでカードがドラッグできない

A. **長押し → ドラッグ** の手順で動かせます。タップしてもカードが開くだけです。
